# TreeVision 🌳

**AI-powered tree species detection and crown segmentation dashboard.**  
Built on **YOLOv8** (Ultralytics) + **FastAPI** backend + **React + Vite** frontend.

Detects and classifies **33 Indian tree species** from drone/aerial orthomosaic imagery, renders annotated detection images, telemetry charts, a human-in-the-loop review queue, and exports results to CSV / GeoPackage.

## Setup — First Time

### 1. Clone the repository

```bash
git clone https://github.com/your-org/TreeVision.git
cd TreeVision
```

### 2. Set up the Python backend

```bash
cd backend

# Create virtual environment
python -m venv venv

# Activate — Windows
venv\Scripts\activate

# Activate — macOS / Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### 3. Place model weights

Copy `TreeVision_best.pt` into the `backend/weights/` folder:

```
backend/weights/TreeVision_best.pt
```

### 4. Add your drone images

Drop `.jpg`, `.jpeg`, `.png`, or `.tif` orthomosaic tiles into `data/images/`:

```
data/images/patch_0.jpg
data/images/patch_1.jpg
...
```

### 5. Set up the frontend (first time only)

```bash
cd frontend
npm install
```

## Running the Application

### ▶ Start the backend server

Open a terminal in `backend/` and run:

```bash
# Windows (PowerShell)
cd backend
$env:DATA_DIR="C:\path\to\TreeVision\data"
$env:WEIGHTS_PATH="C:\path\to\TreeVision\backend\weights\TreeVision_best.pt"
venv\Scripts\python -m uvicorn server:app --host 127.0.0.1 --port 8001

# macOS / Linux
cd backend
DATA_DIR=/path/to/TreeVision/data \
WEIGHTS_PATH=/path/to/TreeVision/backend/weights/TreeVision_best.pt \
./venv/bin/uvicorn server:app --host 127.0.0.1 --port 8001
```

Open **http://localhost:8001** — the React dashboard is served automatically.

You should see this in the terminal:
```
==================================================
  TreeVision — starting up
==================================================
  Weights : backend/weights/TreeVision_best.pt
  Data dir: data
  Outputs : data/outputs
[predict] ✅ Model loaded — TreeVision_best.pt
[predict]    33 classes: ['African Tulip Tree', 'Agathi', ..., 'White mulberry']
  Species : 34 classes loaded from weights (incl. Other)
==================================================
```

> **Tip:** You can also set variables in `backend/.env` and the server will pick them up automatically via `python-dotenv`.

---

### ▶ Frontend development server (hot-reload)

If you're modifying the React source, run Vite's dev server in a **second terminal**:

```bash
cd frontend
npm run dev
```

Open **http://localhost:5173** — changes to `.jsx` / `.css` files reload instantly.

> In dev mode, Vite proxies `/api/*` requests to `http://localhost:8001`.  
> Make sure the backend is already running before starting the dev server.

---

### ▶ Build frontend for production

When you're done making frontend changes, compile them into the backend's static folder:

```bash
cd frontend
npm run build
```

This outputs the compiled bundle to `backend/static/` so the FastAPI server serves it at `http://localhost:8001`.

## Using the Dashboard

### Step 1 — Select an image

**Option A — Image Gallery (server-side tiles)**
1. Click **"Browse local image library"** in the left sidebar
2. A modal shows all tiles from `data/images/` as thumbnail cards
3. Use the search bar or TIFF/JPEG filter chips to find your tile
4. Click **Select** on a card

**Option B — Direct upload**
1. Drag and drop a `.jpg`, `.tif`, or `.png` onto the upload zone
2. Or click the upload zone to open a file picker

---

### Step 2 — Run detection

1. Adjust the **Confidence Gates** sliders if needed:
   - **Auto-Accept Lock** — detections above this are trusted automatically (default 80%)
   - **Minimum Gate** — detections below this are ignored entirely (default 35%)
2. Click **Run Forest Scan**

The system runs YOLOv8 inference (IoU threshold 0.85, up to 1000 detections) and returns:
- A **YOLO-annotated image** with boxes drawn natively by the model
- Full detection features for charts and review

---

### Step 3 — Explore results

| Tab | What you see |
|-----|-------------|
| **Detection View** | The uploaded image with colored bounding boxes drawn by YOLO. Named species get solid-colored boxes; uncertain detections get dashed grey boxes labeled "Tree (uncertain)". Zoom/pan with scroll wheel. |
| **Telemetry** | Species donut chart, confidence score histogram, mean crown area per species bar chart |
| **Review** | All low-confidence detections queued for manual correction. Pick the correct species → click Correct, or click Accept to confirm. |
| **Export** | Download raw CSV, validated CSV, or open the auto-saved GeoPackage path |

---

## Detection Behaviour

| Confidence | Label | Box Style |
|-----------|-------|-----------|
| ≥ 45% | Named species (e.g. `Mango · 73%`) | Solid colored border |
| < 45% | `Tree (uncertain)` | Dashed grey border |
| < Minimum Gate | Ignored | Not shown |

Species with confidence ≥ Auto-Accept threshold → `AUTO_ACCEPTED`  
Species below Auto-Accept but above Minimum → `REVIEW_REQUIRED`  
Manually corrected in Review tab → `MANUALLY_VERIFIED`

## Outputs (auto-saved)

Every prediction automatically saves to `data/outputs/`:

| File | Contents |
|------|----------|
| `<tile>_<timestamp>.csv` | All detections: species, confidence, bbox, crown area, circularity |
| `<tile>_<timestamp>.gpkg` | Crown polygons in EPSG:4326 — only for georeferenced GeoTIFFs |
| `corrections.csv` | Appended log of all manual validations from the Review queue |

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Serve the React SPA |
| `GET` | `/api/status` | Model health, species list, confidence thresholds |
| `GET` | `/api/images` | List all images in `data/images/` |
| `GET` | `/api/images/{filename}/thumb` | JPEG thumbnail (280×180px) for gallery |
| `GET` | `/api/images/{filename}/full` | Full-resolution image (TIFF→JPEG on the fly) |
| `POST` | `/api/predict` | Predict from a multipart file upload |
| `POST` | `/api/predict-by-name` | Predict from a file already in `data/images/` |
| `POST` | `/api/annotated-image/by-name` | YOLO-annotated JPEG for gallery file |
| `POST` | `/api/annotated-image/upload` | YOLO-annotated JPEG for uploaded file |
| `POST` | `/api/validate` | Submit a manual species correction |
| `POST` | `/api/export/csv` | Stream detections as CSV |

## Environment Variables

Set in `backend/.env` or as shell env vars before starting the server:

```env
# Path to the YOLO weights file
WEIGHTS_PATH=C:\Users\you\TreeVision\backend\weights\TreeVision_best.pt

# Root data directory (must contain an images/ subfolder)
DATA_DIR=C:\Users\you\TreeVision\data
```

Confidence thresholds are set in `backend/species.py`:

```python
CONF_MIN  = 0.35   # detections below this are ignored
CONF_AUTO = 0.80   # detections above this are auto-accepted
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `Server offline` in nav bar | Backend uvicorn is not running |
| `Model not loaded` status | Check `backend/weights/TreeVision_best.pt` exists |
| Port already in use | Change `--port 8001` to another port (e.g. `8002`) |
| Gallery empty | Drop images into `data/images/` and refresh the page |
| TIFF images fail to load | Install rasterio: `pip install rasterio` |
| GeoPackage not created | Install geopandas: `pip install geopandas` |
| Boxes not showing after scan | Hard-refresh the browser with `Ctrl+Shift+R` |
| Frontend changes not reflected | Run `npm run build` from `frontend/` directory |
| `npm run dev` API errors | Make sure the backend is running before starting the dev server |
